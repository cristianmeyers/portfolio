01 · vue d'ensemble

À quoi sert WakeJS ?

Dans un environnement multi-VLAN comme une cybersalle ou un établissement scolaire, allumer manuellement chaque poste est une perte de temps. WakeJS lit la configuration DHCP ISC, résout automatiquement les adresses broadcast par VLAN et envoie le paquet magique - le tout depuis un navigateur.

Wake-on-LAN

Envoi de paquets magiques UDP port 9 vers les machines cibles, avec résolution automatique du broadcast par VLAN.

Ping ICMP

Vérification instantanée du statut de chaque poste depuis l'interface. Retour visuel en temps réel.

Arrêt distant

Extinction des postes via SSH (sshpass + sudo shutdown) sans accès physique.

Auth Active Directory

Authentification JWT couplée à LDAP/AD. Seuls les comptes du domaine peuvent accéder à l'interface.

Multi-VLAN

Gestion de plusieurs plages IP avec calcul automatique de l'adresse broadcast selon l'IP de chaque hôte.

Config DHCP ISC

Parsing direct du fichier dhcp-template.conf - pas de base de données séparée à maintenir.

03 · architecture réseau

Deux VLANs, un seul outil

WakeJS résout automatiquement l'adresse broadcast à envoyer selon l'IP de l'hôte cible. Pas besoin de configuration manuelle par VLAN.

VLAN Plage IP Broadcast Usage
Gestion 172.18.53.x → 172.18.59.x 172.18.60.255 Postes admin, salles de gestion
Enseignement 172.18.240.x → 172.18.247.x 172.18.240.255 Salles de cours, postes apprenants

05 · installation

Démarrage rapide

Prérequis avant installation - chaque point est indispensable au bon fonctionnement de WakeJS.

Node.js ≥ 18 - runtime backend obligatoire.

sshpass installé sur le serveur WakeJS pour l'arrêt distant via SSH.

Accès réseau UDP port 9 vers les machines cibles + Wake-on-LAN activé dans le BIOS.

Accès LDAP au contrôleur de domaine AD et compte de service avec droits sudo shutdown.

# 1. Cloner le dépôt
$ git clone https://github.com/cristianmeyers/wakejs.git
$ cd wakejs/backend

# 2. Installer les dépendances
$ npm install

# 3. Configurer (VLAN, AD, SSH)
$ nano backend/config/config.json

# 4. Lancer le serveur
$ bash scripts/wakejs.sh

# API disponible sur http://0.0.0.0:3000
