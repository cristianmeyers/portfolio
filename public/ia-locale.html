01 · vue d'ensemble

Un LLM chez soi, sans compromis

Zéro cloud, zéro API payante. L'inférence tourne localement sur GPU, les données ne quittent pas le réseau, et l'interface est accessible depuis n'importe quel navigateur.

Inférence GPU locale

llama.cpp compilé avec support CUDA pour deux cartes NVIDIA (P2000 + T400) — les couches du modèle sont réparties sur les deux GPU.

Gemma 4 · 8B

Modèle Google Gemma 4 8 milliards de paramètres — bon équilibre performance/VRAM pour du matériel prosumer.

Open WebUI

Interface web type ChatGPT, self-hostée via Docker. Historique des conversations, gestion des modèles, accès multi-utilisateurs.

n8n · Automatisation

Orchestrateur de workflows no-code/low-code. Permet de brancher le LLM sur des pipelines automatisés : webhooks, APIs, fichiers.

100 % privé

Aucune donnée ne quitte le réseau local. Pas de compte cloud, pas d'abonnement, pas de télémétrie externe.

Stack Docker isolée

n8n et Open WebUI déployés en containers sur un nœud Proxmox dédié — séparation claire entre inférence et services applicatifs.

02 · architecture

Deux nœuds, un seul lab

Le serveur GPU tourne sur Ubuntu minimal pour maximiser les ressources disponibles pour l'inférence. Les services applicatifs sont isolés sur un nœud Proxmox Docker séparé.

03 · détail des nœuds

Configuration des serveurs

Chaque nœud a un rôle précis. La séparation évite que les services applicatifs concurrencent l'inférence GPU en mémoire et en CPU.
